Brik was built for CMS.

The problem with stylesheet is that you have to find the style and write additional classes to override the current class, or worst, !important it. What are we animals? There's got to be a better way!

Styling should be simple. Extending existing styles should be even simpler. Imagine looking at the paragraph and seeing: class="padding-50". What would that tell you? Or what if you saw a heading with: class="size-4em"? Wouldn't this be easier to understand? Wouldn't this be easier to modify than to go hunting for class="my-new-heading-V2_Final"?

You're probably thinking: "that's way too many classes to write out!" And, yes, you're right. But if you think of all the little minor adjustments that you do on a daily basis, there's not that many.

If you have a pre-defined unit of measurement for padding or margin (5s, 10s, or 12s), you can make those in to a class. This means that even if you have defined styles that uses a stylesheet, you can always override them using Brik.

Let say you have a stylesheet with a p {margin-bottom:20px;} but want the last one to be p {margin-bottom:0;}. Of course you can use the :last-child property, but what if this was a author in CMS that doesn't know or have limited access?
